# Success Criteria & Deliverable Checklist

## Success Criteria
The project is complete when:
- [ ] All custom fields are visible on the Lead layout
- [ ] Validation rules correctly block improper entries
- [ ] The Lightning App loads without errors and displays the Leads tab
- [ ] The Record-Triggered Flow updates `Lead_Score__c` automatically on every create/update
- [ ] `AI_Summary__c` is populated with a meaningful AI-generated insight
- [ ] Screenshots and documentation are clear, professional, and comprehensive
- [ ] GitHub repository is structured and contains all deliverables

## Final Submission Checklist
- [ ] Developer Org created and accessible
- [ ] Custom Lead fields (Lead Score, AI Summary, Industry Type, Engagement Level)
- [ ] Validation rule implemented and tested
- [ ] Lightning App built with proper tabs
- [ ] Profiles (Sales Rep, Sales Manager) configured
- [ ] Role hierarchy established
- [ ] Test users created and assigned
- [ ] Record-Triggered Flow active
- [ ] Prompt Builder template created
- [ ] AI Summary field populating correctly
- [ ] End-to-end testing performed (3+ test leads)
- [ ] Screenshots captured for each milestone
- [ ] GitHub repository initialized with README, docs, and screenshots
- [ ] Project documentation written (Abstract, Architecture, Testing, Conclusion)
- [ ] Final verification completed

## Project Deliverables
| Deliverable | Format | Location in this repo |
|---|---|---|
| Salesforce Org URL | URL | Add to `README.md` before submitting |
| Screenshots | Image folder | `Screenshots/` |
| GitHub Repository | Repo | This repo |
| Project Documentation | Word/PDF | `Documentation/Project_Report.docx` |
| Test Evidence | Screenshots/Logs | `Testing/Test_Plan_and_Results.md`, `Screenshots/07_Testing/` |
| Commit History | Git log | `git log --oneline` |

Only check a box once it's true in the live org — a reviewer will very likely log in and verify these directly.
